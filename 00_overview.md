# 00 — Overview

## Vision
Build the first intent-driven social execution layer where users express what they want to do or talk about, and the system connects them with the right people in real time.

## Product thesis
Social software is still optimized around browsing:
- browse feeds
- browse groups
- browse events
- browse profiles

This product is optimized around intent:
- “I want to talk about the match”
- “I want to play table tennis tonight”
- “I want a chill Valorant duo now”
- “I want to meet people into AI and design in my city”

The user does not search manually. The system interprets intent, routes requests, and creates opt-in human connections.

## What this product is
- A conversational social routing system
- A real-time coordination layer for people
- An intent graph + matching engine + trust system
- A human-to-human messaging product with AI-assisted routing and safety

## What this product is not
- Not a feed-first social network
- Not an AI companion app
- Not an AI chat app where the model talks for the user
- Not a dating app first
- Not a community browser or directory clone

## Product invariant
AI may:
- parse
- classify
- enrich
- rank
- summarize
- coordinate
- enforce policy
- propose actions

AI may not:
- send live chat messages as the user by default
- simulate social relationships
- create hidden, irrevocable actions on behalf of the user
- silently contact users outside explicit product flows

## North-star outcome
A user writes one sentence expressing a social need and reaches a high-quality human interaction quickly and safely.

## Primary KPI
Time from intent creation to successful connection.

## Secondary KPIs
- request acceptance rate
- first-message rate after match
- conversation completion rate
- repeat interaction rate
- weekly active intent creators
- trust-adjusted successful connection rate
- report/block rate per 1,000 interactions
