# 13 — Safety, Trust, and Moderation

## Principle
Safety is not a later moderation layer. It is built into routing, consent, messaging, and trust scoring.

## Safety controls
- opt-in requests only
- no unsolicited DM creation
- report
- block
- mute topic/requester
- request cooldowns
- recipient saturation protection
- account-level rate limits
- abuse heuristics
- moderation review queue

## Trust features
- account age
- verified login
- successful connection rate
- report ratio
- block ratio
- moderation actions
- spam-like sending behavior
- recipient acceptance patterns

## Moderation workflow
1. flag generated by user or system
2. moderation case created
3. triage decision:
   - dismiss
   - warn
   - restrict
   - suspend
   - ban
4. trust recalculation
5. appeal path if needed

## AI safety usage
Use model classification for:
- sexual solicitation
- harassment
- hate/violence
- scams/fraud
- self-harm escalation
- illegal/risky requests

Never rely on model classification alone for terminal punitive action without deterministic thresholds or human review on severe cases.

## Abuse scenarios
- spam intents
- mass targeting of one user cohort
- abusive or manipulative language
- account farming / fake profiles
- group infiltration
- catfishing / misrepresentation
- minors / age-sensitive misuse if product scope expands

## Launch guardrails
- conservative rate limits
- manual review tooling
- seeded moderation dashboard
- logging and auditability from day one
